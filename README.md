# 🗂️ skill-inventory

> Manage, audit, and clean your [Claude Code](https://claude.ai/code) skills — global, local, and plugins — from a single CLI.

---

## ✨ Features

- 📦 **Inventories all skill sources** — `~/.claude/skills/`, project-local skills, and installed plugins
- 🏷️ **Categorizes automatically** — SEO, LinkedIn, Testing, Vercel, SDD, Security, and more
- 🔍 **Local duplicate detection** — compares skill descriptions with `difflib`, zero API needed
- 🔌 **Cross-scope detection** — spots global skills shadowed by an installed plugin
- 🧹 **Zero-API cleanup** — run `audit` or `prune` once, then `clean` or `clean --yes` to apply
- 🔎 **Gap analysis** — given a GitHub repo URL, shows which skills you have, which to install, and which tech areas have no coverage
- 🌱 **Project-aware pruning** — finds global skills irrelevant to any of your local projects
- 💾 **Safe deletions** — every removed skill is backed up before deletion

### 🔍 `audit` — Local duplicate detection

```bash
skill-inventory audit
```

Reads every skill's YAML frontmatter description and compares all pairs using `difflib.SequenceMatcher`. No API call, no tokens, instant results.

Detects:
- 🔁 **Duplicates** — two global/local skills with very similar descriptions or names
- 🔌 **Shadowed** — a global/local skill that duplicates an installed plugin skill
- ⚠️ **Stubs** — skills under 100 bytes with no real content

Results are saved to `~/.claude/skill-inventory-report.json`.

```
▸ Auditing skills  (local analysis — no API)
  Comparing 83 global/local skills · cross-checking 122 plugin skills…

  2 duplicate(s), 22 shadowed by plugin, 0 warning(s).

  Duplicate skills (2 group(s))

  [1] Duplicate  Descriptions 89% similar
      Keep:    ~/.claude/skills/prfeature/SKILL.md
      Remove:  ~/.claude/skills/prfix/SKILL.md

  Shadowed by plugin (22 group(s))

  [1] Shadowed  Descriptions 100% similar
      Plugin:  agent-skills:api-and-interface-design
      Remove:  ~/agent-skills/skills/api-and-interface-design/SKILL.md  (your global copy)

  ✓  Report saved → ~/.claude/skill-inventory-report.json
  ·  Run skill-inventory clean to apply changes
```

### 🔎 `match` — Gap analysis against a GitHub repo

```bash
skill-inventory match https://github.com/owner/repo
```

Fetches the repo's metadata and dependency files from GitHub, extracts the tech stack, and classifies every skill into three categories:

| Label | Meaning |
|-------|---------|
| `[✓ HAVE]` | Skill already in your global/local collection |
| `[↓ INSTALL]` | Skill available in an installed plugin — not yet global |
| `[✗ GAP]` | Tech area with no skill at all, even in plugins |

Skills in the `[↓ INSTALL]` section are also cross-referenced against your local projects. Any skill marked `★ also useful for YOUR projects` is relevant not just to the GitHub repo but to your own codebase — install those first.

```

    [↓ INSTALL]  Skill exists in a plugin but NOT yet in your global skills
```

### 🌱 `prune` — Remove skills unused by your projects

```bash
skill-inventory prune
```

Scans all detected local projects, extracts their tech stacks (from `package.json`, `requirements.txt`, `go.mod`, `pyproject.toml`, `Cargo.toml`, and file extensions), then scores every global skill against the combined keyword set.

| Label | Meaning |
|-------|---------|
| `[✗ UNUSED]` | Score 0 across all projects — safe to remove |
| `[? LOW]` | Score 1 — marginally relevant, review before removing |
| `[✓ USED]` | Score > 1 — actively useful, keep |

Saves `UNUSED` skills to `~/.claude/skill-inventory-report.json` so `clean` can process them.

```
▸ Prune — finding skills unused across all your projects
  · Scanning 11 project(s) for tech stack…
  Combined stack: typescript, react, nextjs, prisma, python …

  Legend
    [✗ UNUSED]    Score 0 across all your projects — safe to remove
    [? LOW]       Score 1 — marginally relevant, review before removing
    [✓ USED]      Score > 1 — actively useful, keep

  [✗ UNUSED]  8 skill(s) — zero relevance to your projects
    ✗  landing-page-generator  (global)
    ✗  linkedin-messages-agent  (global)
    ...

  [? LOW]  22 skill(s) — low relevance (score = 1)
    ...

  [✓ USED]  30 skill(s) — actively relevant
    ...

  ✓  Report saved → ~/.claude/skill-inventory-report.json
  ·  Run skill-inventory clean to remove unused skills (with backup)
```

## 📥 install-skills

A companion bash utility that copies plugin skills into `~/.claude/skills/` so they become globally available across all projects.

### Installation

The script is included in this repo. Copy it to any directory in your `PATH`:

```bash
cp install-skills ~/.local/bin/install-skills
chmod +x ~/.local/bin/install-skills
```

### Commands

```bash
# Interactive — choose which skills to install
install-skills

# Install specific skills by name
install-skills vercel-cli shadcn

# Install ALL available plugin skills not yet in globals
install-skills --all

# List what's available without installing
install-skills --list

# Install skills recommended by skill-inventory match
install-skills --from-match github.com/owner/repo
```

### `--from-match` — Install gap recommendations automatically

The most powerful mode. Runs `skill-inventory match` internally, extracts the `[↓ INSTALL]` candidates, and installs each one that exists in your plugin cache:

```bash
install-skills --from-match github.com/mattpocock/skills
```

```
Running skill-inventory match github.com/mattpocock/skills…

Installing recommended skills…

  ✓ systematic-debugging
  ✓ next-forge
  ~ shadcn — already installed, skipping

Done — 2 installed, 1 already present
```

## 🔄 Recommended workflow

```bash
# Step 1 — find and remove duplicates / shadowed skills
skill-inventory audit
skill-inventory clean --yes

# Step 2 — find skills irrelevant to your projects
skill-inventory prune
skill-inventory clean --yes

# Step 3 — explore a repo for gaps and install what's missing
skill-inventory match github.com/owner/repo
install-skills --from-match github.com/owner/repo

# Step 4 — re-audit to confirm
skill-inventory audit
```

## 📁 Where skills are scanned

| Source | Path |
|--------|------|
| 🌐 Global | `~/.claude/skills/` |
| 🔌 Plugins | `~/.claude/plugins/cache/` |
| 📁 Local | `<project>/skills/` |
| 🗂️ Projects | `~/` (folders with `.git`, `CLAUDE.md`, or `package.json`) |

## 🏗️ Architecture

```
skill-inventory <cmd>
       │
       ▼
 build_snapshot()
       ├── ~/.claude/skills/          → global skills (SKILL.md or flat .md)
       ├── ~/.claude/plugins/cache/   → plugin skills (latest version only)
       └── ~/*/skills/                → local project skills
       │
       ├── scan   → print summary
       ├── list   → categorized display (by category / project / namespace)
       ├── audit  → difflib comparison → ~/.claude/skill-inventory-report.json
       ├── clean  → read report → delete + backup (interactive or --yes bulk)
       ├── match  → GitHub API + local project scan → gap analysis
       └── prune  → local project scan → orphan detection → report
```

### How duplicate detection works

1. Extract `description:` from each skill's YAML frontmatter (handles inline values and `>` / `|` block scalars)
2. Normalize skill names (strip namespace prefix, replace `-`/`_` with spaces)
3. Compare every pair with `difflib.SequenceMatcher`
4. Flag as **duplicate** if description similarity ≥ 80%, name similarity ≥ 90%, or both ≥ 70%
5. Cross-check each global/local skill against every plugin skill — flag as **shadowed** on the same thresholds (plugin always wins)

### How gap analysis works

1. Fetch repo metadata and dependency files via GitHub public API (no auth needed)
2. Extract tech keywords from `package.json`, `requirements.txt`, `go.mod`, `pyproject.toml`, `Cargo.toml`, topics, and language
3. Score every skill against those keywords using `TECH_ALIASES` semantic mapping
4. Classify: own skills that score > 0 → `HAVE`, plugin skills not covered by own → `INSTALL`, tech areas with no coverage anywhere → `GAP`
5. Cross-reference `INSTALL` candidates against local project keywords → mark `★` priority

### How prune works

1. Detect all local projects under `~/`
2. Extract tech keywords from each project's dependency files and file extensions
3. Score every global/local skill against the combined keyword union
4. Orphaned skills (score = 0) → saved to report for `clean` to process
